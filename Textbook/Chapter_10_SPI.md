## 10.1 Concept Overview

### What Is SPI?

SPI (Serial Peripheral Interface) is a **synchronous, full-duplex** communication protocol using 4 wires. It's fast, simple, and widely used for communicating with sensors, displays, flash memory, and SD cards.

```
  SPI Bus Wiring:
  
  ┌──────────┐                         ┌──────────┐
  │  MASTER  │    SCLK (Clock) ───────→│  SLAVE   │
  │  (STM32) │    MOSI (Master Out) ──→│ (Sensor, │
  │          │    MISO (Master In) ←───│  Flash)  │
  │          │    CS/SS (Chip Select)──→│          │
  └──────────┘         (Active LOW)    └──────────┘
  
  SCLK: Clock signal generated by master
  MOSI: Data from master to slave (Master Out, Slave In)
  MISO: Data from slave to master (Master In, Slave Out)
  CS:   Chip select — LOW to activate slave
  
  
  Multiple Slaves:
  
  ┌──────┐            ┌────────┐
  │      │──── SCLK ─→│ Slave1 │
  │      │──── MOSI ─→│        │
  │Master│←─── MISO ──│  CS1←──┼── CS1 (PA4)
  │      │            └────────┘
  │      │            ┌────────┐
  │      │──── SCLK ─→│ Slave2 │
  │      │──── MOSI ─→│        │
  │      │←─── MISO ──│  CS2←──┼── CS2 (PB0)
  └──────┘            └────────┘
  
  Only one CS is LOW at a time — selects which slave to talk to.
```

### SPI vs UART vs I2C Comparison

| Feature | SPI | UART | I2C |
|---------|-----|------|-----|
| Wires | 4 (+ CS per slave) | 2 | 2 |
| Speed | Up to 50+ MHz | Up to ~1 MHz | Up to 3.4 MHz |
| Duplex | Full duplex | Full duplex | Half duplex |
| Clock | Synchronous (master) | Asynchronous | Synchronous (master) |
| # Devices | Limited by CS pins | Point-to-point | Up to 127 (addressed) |
| Complexity | Simple | Simple | Medium |

## 10.2 STM32 Hardware Internals

### SPI Data Exchange

```
  SPI operates as two connected shift registers:
  
  MASTER                              SLAVE
  ┌────────────────────┐              ┌────────────────────┐
  │ Shift Register     │    MOSI     │ Shift Register     │
  │ [7][6][5][4][3][2][1][0] ──────→ [7][6][5][4][3][2][1][0] │
  │                    │    MISO     │                    │
  │ [7][6][5][4][3][2][1][0] ←────── [7][6][5][4][3][2][1][0] │
  └────────────────────┘    SCLK     └────────────────────┘
                            ──────→
  
  With each clock cycle, one bit shifts out on MOSI,
  and one bit shifts in on MISO.
  After 8 clocks, both have exchanged one byte.
  
  This means: to RECEIVE data, you must SEND data (even dummy bytes)!
```

### SPI Clock Polarity and Phase (CPOL/CPHA)

```
  SPI has four modes based on CPOL and CPHA:
  
  Mode 0 (CPOL=0, CPHA=0) — Most common
  SCLK idle LOW, sample on RISING edge
  
  SCLK: ──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──
           └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘
  MOSI: ═══╤════╤════╤════╤════╤════╤════╤════╤═══
           D7   D6   D5   D4   D3   D2   D1   D0
          ↑    ↑    ↑    ↑    ↑    ↑    ↑    ↑
        Sample on rising edge
  
  Mode 3 (CPOL=1, CPHA=1) — Also common
  SCLK idle HIGH, sample on RISING edge
  
  Check the datasheet of your slave device to determine which mode!
```

### STM32 SPI Registers

```
  SPI1 Base: 0x40013000 (APB2)
  SPI2 Base: 0x40003800 (APB1)
  
  Register  Offset  Key Bits
  ────────  ──────  ──────────────────────────────────
  CR1       0x00    BR[2:0]: Baud rate (f_PCLK / 2..256)
                    MSTR: Master mode
                    CPOL, CPHA: Clock polarity/phase
                    SPE: SPI enable
                    SSM, SSI: Software slave management
                    DFF: Data frame format (8 or 16 bit)
  
  CR2       0x04    TXEIE, RXNEIE: Interrupt enables
                    TXDMAEN, RXDMAEN: DMA enables
  
  SR        0x08    TXE: TX buffer empty
                    RXNE: RX buffer not empty
                    BSY: SPI busy flag
  
  DR        0x0C    Data register (read = RX, write = TX)
```

## 10.3 Step-by-Step Implementation

### SPI Master Driver

```c
/******************************************************************************
 * File:    spi_driver.h / spi_driver.c
 * Brief:   SPI1 master driver for STM32F401
 *
 * SPI1 pins (AF5):
 *   PA5 = SCK    (also LD2! Use SPI2 or external SCK if LED needed)
 *   PA6 = MISO
 *   PA7 = MOSI
 *   PA4 = CS (manual GPIO control)
 *****************************************************************************/

#include <stdint.h>

/* -- Type definitions -- */
typedef struct {
    volatile uint32_t CR1;
    volatile uint32_t CR2;
    volatile uint32_t SR;
    volatile uint32_t DR;
    volatile uint32_t CRCPR;
    volatile uint32_t RXCRCR;
    volatile uint32_t TXCRCR;
    volatile uint32_t I2SCFGR;
    volatile uint32_t I2SPR;
} SPI_TypeDef;

#define SPI1    ((SPI_TypeDef *)0x40013000UL)
#define SPI2    ((SPI_TypeDef *)0x40003800UL)

/* SR bits */
#define SPI_SR_TXE      (1UL << 1)
#define SPI_SR_RXNE     (1UL << 0)
#define SPI_SR_BSY      (1UL << 7)

/* CR1 bits */
#define SPI_CR1_SPE     (1UL << 6)
#define SPI_CR1_MSTR    (1UL << 2)
#define SPI_CR1_SSM     (1UL << 9)
#define SPI_CR1_SSI     (1UL << 8)

void spi1_init(void)
{
    /* Enable clocks */
    (*(volatile uint32_t *)0x40023830UL) |= (1UL << 0);  /* GPIOA */
    (*(volatile uint32_t *)0x40023844UL) |= (1UL << 12); /* SPI1 */

    /* PA5 (SCK), PA6 (MISO), PA7 (MOSI) as AF5 */
    volatile uint32_t *moder = (volatile uint32_t *)0x40020000UL;
    volatile uint32_t *afrl  = (volatile uint32_t *)0x40020020UL;

    /* Set PA5, PA6, PA7 to AF mode (MODER = 10) */
    *moder &= ~((3UL << 10) | (3UL << 12) | (3UL << 14));
    *moder |=  ((2UL << 10) | (2UL << 12) | (2UL << 14));

    /* Set AF5 for PA5, PA6, PA7 */
    *afrl &= ~((0xFUL << 20) | (0xFUL << 24) | (0xFUL << 28));
    *afrl |=  ((5UL << 20) | (5UL << 24) | (5UL << 28));

    /* PA4 as GPIO output for CS (manual control) */
    *moder &= ~(3UL << 8);
    *moder |=  (1UL << 8);
    /* Set CS high (deselect) */
    (*(volatile uint32_t *)0x40020018UL) = (1UL << 4); /* BSRR set PA4 */

    /* Configure SPI1:
     * - Master mode
     * - Clock prescaler = /16 (for ~1 MHz at 16 MHz APB2)
     * - CPOL=0, CPHA=0 (Mode 0)
     * - 8-bit data frame
     * - MSB first (default)
     * - Software slave management (SSM=1, SSI=1)
     */
    SPI1->CR1 = SPI_CR1_MSTR        /* Master mode */
              | SPI_CR1_SSM          /* Software slave management */
              | SPI_CR1_SSI          /* Internal slave select */
              | (3UL << 3);          /* BR = 011 → f/16 */

    /* Enable SPI */
    SPI1->CR1 |= SPI_CR1_SPE;
}

/* Chip Select control */
void spi1_cs_low(void)
{
    (*(volatile uint32_t *)0x40020018UL) = (1UL << (4 + 16)); /* BSRR reset */
}

void spi1_cs_high(void)
{
    (*(volatile uint32_t *)0x40020018UL) = (1UL << 4); /* BSRR set */
}

/*
 * spi1_transfer() — Send and receive one byte simultaneously
 */
uint8_t spi1_transfer(uint8_t tx_data)
{
    /* Wait until TX buffer is empty */
    while (!(SPI1->SR & SPI_SR_TXE));

    /* Write data to send */
    SPI1->DR = tx_data;

    /* Wait until RX buffer has data */
    while (!(SPI1->SR & SPI_SR_RXNE));

    /* Read and return received data */
    return (uint8_t)(SPI1->DR);
}

/*
 * spi1_write_reg() — Write to a slave register
 * Common pattern: send register address, then data
 */
void spi1_write_reg(uint8_t reg_addr, uint8_t value)
{
    spi1_cs_low();
    spi1_transfer(reg_addr);       /* Send register address */
    spi1_transfer(value);          /* Send value */
    spi1_cs_high();
}

/*
 * spi1_read_reg() — Read from a slave register
 * Common pattern: send address with READ bit, receive data
 */
uint8_t spi1_read_reg(uint8_t reg_addr)
{
    uint8_t value;
    spi1_cs_low();
    spi1_transfer(reg_addr | 0x80U);  /* MSB=1 for read (device-specific!) */
    value = spi1_transfer(0xFFU);     /* Send dummy byte, receive data */
    spi1_cs_high();
    return value;
}
```

## 10.4 Common Mistakes & Debugging

| Mistake | Symptom | Fix |
|---------|---------|-----|
| Wrong SPI mode (CPOL/CPHA) | Slave returns wrong data | Check slave datasheet for mode |
| CS not toggling properly | Slave doesn't respond | Verify CS goes LOW then HIGH |
| Clock too fast for slave | Corrupt data | Reduce SPI clock speed |
| MOSI/MISO swapped in wiring | All zeros or all ones | Verify physical connections |
| Forgetting to read DR | RXNE never clears | Always read after every write |
